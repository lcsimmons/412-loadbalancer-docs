◆ addNewRequests()

void LoadBalancer::addNewRequests ( )

Adds new random requests to the request queue.

A random number of new requests (between 1 and 5) are generated and added to the request queue. Each request is assigned random IP addresses, a random job type ('P' for processing or 'S' for streaming), and a random amount of time needed to process the request.

◆ distributeRequest()

void LoadBalancer::distributeRequest ( )

Assigns available web servers to process requests from the queue.

Loops through the web servers and assigns a request from the queue to any server that is not currently busy. This helps balance the load by processing requests as soon as servers become available.

◆ isUnderServerToRequestThreshold()

bool LoadBalancer::isUnderServerToRequestThreshold ( )

Checks if the current number of web servers is overwhelmed by the request load.

This method assumes that a single server can handle up to 100 requests. If the current request queue length divided by the number of servers exceeds this threshold, the system is considered to be under the threshold (overloaded).

Returns
true If the current server-to-request ratio is overloaded (i.e., more than 100 requests per server).
false If the current server-to-request ratio is within acceptable limits.

◆ manageServers()

void LoadBalancer::manageServers ( )

Dynamically manages the allocation and deallocation of web servers.

This method adds new servers if the current server-to-request ratio is overwhelmed. It removes unnecessary servers when the load decreases. A minimum of 10 servers will always remain active, and only idle servers will be removed.

◆ tick()

void LoadBalancer::tick ( )

Advances the simulation by one clock cycle.

This method communicates to all web servers that one clock cycle has passed. Each web server will decrement its remaining processing time for the current request, if any.
